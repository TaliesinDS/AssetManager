the purpose of this repo is to create tools to make managing assets for my unity game project easier.
assets i handle include 3d models, textures, sound etc.
tools i use: engine: unity, modelling: blender, fusion 360 for easy stuff. i have access to substance designer/painter/sampler but only use those when really needed, i prefer to stick to blender and fusion.
i also have photoshop, illustrator etc.

currently the focus is on managing my textures, i have a folder with textures which each are in their own subfolder, i want to be able to browse these as thumbnails of materials so i can quickly find a specific road or brick texture i like (integration into blender/unity would be a nice luxury).
i have tried blender asset library but it only sees folders with the .blend in the same folder as the texture map.
i also have folders with no .blend at all or the texture maps are in another folder next to the .blend file.
using the blender asset library and using scripts to make my asset library folders ingestible is something i would consider, i prefer a low friction option instead of building a whole piece of software from scratch.
